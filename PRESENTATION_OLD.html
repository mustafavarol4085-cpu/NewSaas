⚙️Company GPT - Technical Build

Knowledge Base + Golden Calls Architecture

📚 Knowledge Base Pipeline

  • Step 1: Client uploads playbooks, SOPs, FAQs
  • Step 2: Chunk text (512 tokens per chunk with overlap)
  • Step 3: Generate embeddings via OpenAI API
  • Step 4: Store in pgvector (Supabase)
  • Step 5: Query via cosine similarity search
  • Step 6: Inject top 3 chunks into GPT prompt

🏆 Golden Calls Selection

  • Step 1: AI ranks all calls by score (top 5%)
  • Step 2: Extract key moments & techniques
  • Step 3: Store as reference baseline in DB
  • Step 4: Compare new calls against golden baseline
  • Step 5: Highlight what reps do better/worse
  • Step 6: Auto-suggest golden call examples

🗄️pgvector + Golden Calls

Database Schema for Knowledge Base & Baseline Calls

📋 KB Chunks Table (pgvector)

CREATE TABLE kb_chunks (
 id BIGINT PRIMARY KEY,
 document_id UUID,
 chunk_text TEXT,
 chunk_index INT,
 embedding vector(1536),
 source_type TEXT,
 metadata JSONB
);

CREATE INDEX ON kb_chunks
USING ivfflat
(embedding vector_cosine_ops)
WITH (lists = 100);

🏆 Golden Calls Table

CREATE TABLE golden_calls (
 id BIGINT PRIMARY KEY,
 call_id UUID,
 rep_id UUID,
 score DECIMAL,
 techniques JSONB,
 key_moments JSONB,
 objection_responses JSONB,
 talk_ratio DECIMAL,
 created_at TIMESTAMP
);

-- Quick ranking
CREATE INDEX ON golden_calls
(score DESC NULLS LAST);

📊 Golden Calls Selection Logic:

SELECT * FROM golden_calls
WHERE score > (SELECT AVG(score) * 1.25 FROM calls)
ORDER BY score DESC
LIMIT 10; -- Top 10 calls per rep/product
